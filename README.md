# OpenEnv: HR Compliance Review

## Environment Motivation & Description
The **HR Compliance Review** environment simulates an incredibly sensitive, high-stakes task: serving as a corporate HR compliance officer. Agents must triage incoming employee reports, manage IT requests, escalate workplace safety violations, and identify subtle clues related to embezzlement or misconduct while carefully filtering out office gossip. This novel domain challenges an agent's reasoning, logical deduction, and adherence to strict operational protocols.

## Configuration & Specifications
This environment strictly implements the OpenEnv specification, utilizing **Pydantic** to strictly define and validate the `Observation`, `Action`, and `Reward` objects.

### Observation Space
The initial state and observations are returned as an `Observation` struct:
```python
class Report(BaseModel):
    id: str
    sender: str
    subject: str
    body: str
    folder: str
    tags: List[str]
    read: bool
    replied: bool
    reply_body: Optional[str]

class Observation(BaseModel):
    reports: List[Report]
    current_folder: str
```

### Action Space
Agents must output valid JSON matching the `Action` structure:
```python
class Action(BaseModel):
    action_type: Literal['read', 'reply', 'move', 'delete', 'tag']
    item_id: str
    payload: Optional[str] = None
```

### Reward Function
The `Reward` struct returns rich details instead of simple binary boundaries:
- `value`: Final calculated task score (`0.0 - 1.0`).
- `partial_progress`: Fractional points given for moving the state forward (e.g. tagging a single email correctly).
- `penalties`: Deductions for invalid actions or misclassifications.
- `reason`: Textual feedback generated by the task grader at each step.

## Evaluation Tasks
The agent is evaluated across 3 tasks designed to test reasoning logic and resist hallucinations:
1. **Easy: IT Ticket Routing.** The agent must route mundane IT issues ("broken monitor") out of the HR inbox, while ignoring non-IT complaints. (*Score: 1.0 for perfect routing*).
2. **Medium: Workplace Safety Violations.** The agent must identify descriptions of physical safety hazards (spills, unstable shelving), tag them as `safety_hazard`, and issue a reply. It must NOT tag descriptive false-positives describing personal issues (e.g. spilling coffee on a shirt, emotional instability). (*Score: Deductions for false positives or inaction*).
3. **Hard: Whistleblower Escalation.** The agent must find and escalate reports of embezzlement to the 'Confidential' folder and tag them `investigation_required`. It must deliberately ignore simple office gossip. (*Score: 0.5 points for moving correctly, 0.5 points for tagging correctly*).

## Setup & Execution

### Running the Baseline Script
The robust `baseline.py` script attempts the environment using OpenAI or any OpenAI-compatible server.
1. Install dependencies (we recommend `uv` for ultra-fast installs):
    ```bash
    pip install uv
    uv pip install -e .
    ```
2. Set your API credentials:
    ```bash
    # OpenAI key
    export OPENAI_API_KEY="sk-proj-..."
    
    # Optional overrides for local inference / alternative APIs:
    # export OPENAI_BASE_URL="http://localhost:8000/v1" 
    # export OPENAI_MODEL="meta-llama/Llama-3-70b-chat-hf"
    ```
3. Run the evaluation baseline:
    ```bash
    python baseline.py
    ```

### Running the Web Interface
For human evaluation or debugging, a Gradio UI connects directly to the environment logic.
#### Local UI Loop:
```bash
python app.py
```
Open `http://localhost:7860`.

#### Docker / Hugging Face Spaces Engine:
Simply push this directory to a Hugging Face Space using the Docker SDK, or build the image locally:
```bash
docker build -t openenv-hr-compliance .
docker run -p 7860:7860 openenv-hr-compliance
```
